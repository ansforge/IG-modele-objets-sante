Logical Model: Telecommunication - Detailed Descriptions

Draft as of 2025-07-31

Definitions for the Telecommunication logical model.

  • Key Elements Table
  • Differential Elements Table
  • Snapshot Elements Table

Guidance on how to interpret the contents of this table can be foundhere

0. Telecommunication
Définition

Adresse de télécommunication à laquelle une personne ou une organisation peut être contactée (téléphone, fax, e-mail, URL, etc.).

ShortAdresse de télécommunication à laquelle une personne ou une organisation peut être contactée (téléphone, fax, e-mail, URL, etc.).
Control0..*
Is Modifierfalse
Logical ModelInstances of this logical model are not marked to be the target of a Reference
2. Telecommunication.canal
Définition

Code spécifiant le canal ou la manière dont s'établit la communication (téléphone, e-mail, URL, etc.).Quelques exemples de codes:** MOB: Téléphone mobile;** FIX: Téléphone fixe;** EML: Courrier électronique;** URL: Uniform Resource Location;** FTP: File Transfer Protocol;** FAX: Télécopie.

ShortCode spécifiant le canal ou la manière dont s'établit la communication (téléphone, e-mail, URL, etc.).Quelques exemples de codes:** MOB: Téléphone mobile;** FIX: Téléphone fixe;** EML: Courrier électronique;** URL: Uniform Resource Location;** FTP: File Transfer Protocol;** FAX: Télécopie.
Control0..1
BindingThe codes SHOULD be taken from https://mos.esante.gouv.fr/NOS/TRE_R200-CanalCommunication/FHIR/TRE-R200-CanalCommunication?vs
(preferred to https://mos.esante.gouv.fr/NOS/TRE_R200-CanalCommunication/FHIR/TRE-R200-CanalCommunication?vs)
TypeCoding
4. Telecommunication.adresseTelecom
Définition

Valeur de l'adresse de télécommunication dans le format induit par le canal de communication, par exemple un numéro de téléphone, une adresse de courrier électronique, une adresse URL, etc.

ShortValeur de l'adresse de télécommunication dans le format induit par le canal de communication, par exemple un numéro de téléphone, une adresse de courrier électronique, une adresse URL, etc.
Control0..1
Typestring
Primitive ValueThis primitive element may be present, or absent, or replaced by an extension
6. Telecommunication.typeMessagerie
Définition

Type de messagerie électronique rassemblant des acteurs (personne physique, personne morale ou système) identifiés et enregistrés selon des règles qui garantissent leur légitimité à l'utiliser. L'échange de messages obéit à des spécifications fonctionnelles et techniques propres à ce type de messagerie.

ShortType de messagerie électronique rassemblant des acteurs (personne physique, personne morale ou système) identifiés et enregistrés selon des règles qui garantissent leur légitimité à l'utiliser. L'échange de messages obéit à des spécifications fonctionnelles et techniques propres à ce type de messagerie.
Control0..1
BindingThe codes SHOULD be taken from https://mos.esante.gouv.fr/NOS/TRE_R256-TypeMessagerie/FHIR/TRE-R256-TypeMessagerie?vs
(preferred to https://mos.esante.gouv.fr/NOS/TRE_R256-TypeMessagerie/FHIR/TRE-R256-TypeMessagerie?vs)
TypeCoding
8. Telecommunication.utilisation
Définition

Précise l'utilisation du canal de communication (par exemple à des fins professionnelles, privées, etc.).

ShortPrécise l'utilisation du canal de communication (par exemple à des fins professionnelles, privées, etc.).
Control0..1
Typestring
Primitive ValueThis primitive element may be present, or absent, or replaced by an extension
10. Telecommunication.niveauConfidentialite
Définition

Le niveau de confidentialité permet de définir le niveau de restriction de l'accès aux attributs de la classe Telecommunication.

ShortLe niveau de confidentialité permet de définir le niveau de restriction de l'accès aux attributs de la classe Telecommunication.
